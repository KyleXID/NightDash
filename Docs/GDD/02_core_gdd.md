# Layer 2 - Core GDD

## 1. 세계관 및 월드 구조
- 배경: 지구 차원 붕괴 -> 주인공 마계 전이
- 목표: 6지역 정복 후 마왕성과 결전

### 메인 지역 순서
1. 불타는 황야
2. 부패한 숲
3. 그림자 늪 (8~10분)
4. 타락한 성채
5. 절망의 설원
6. 마왕성

### 보너스 맵
- 금지된 도서관: 진화/리롤 특화
- 폐허가 된 투기장: 보스 러시 특화
- 심연의 제단: 리스크 특화

## 2. 핵심 루프
지역 선택 -> 클래스 선택 -> 체크리스트 선택 -> 전투 -> 보스 처치 -> 보상/정복 -> 메타 강화 -> 다음 도전

## 3. 전투/성장 원칙
- 런 내 성장: 무기/패시브/진화/궁극기
- 런 외 성장: 정복의 잔재 기반 클래스 트리
- 클래스마다 시작 무기, 고유 패시브, 궁극기 보유
- 공통 무기 풀은 모든 클래스가 접근 가능

## 4. 패시브 설계 원칙
- 계열: 기본 강화형, 메커니즘 변경형, 리스크-리턴형, 진화 조건형
- 다크 톤 네이밍 유지
- 초보-숙련-고난도 유저를 모두 커버하도록 선택지 분포 설계

## 5. 진화 시스템
### 1단계 일반 진화
- 무기 MAX
- 요구 패시브 보유
- 보스/엘리트 상자 획득

### 2단계 심연 진화
- 진화 무기 + 추가 조건
- 위험도/특정 체크리스트 조건 연동 가능
- 판당 1회 제한

### 진화 밸런스 원칙
- 단순 데미지 배수 중심 금지
- 공격 방식, 범위, 판정, 패턴 변화 우선

## 6. 난이도 체크리스트
- 항목별 위험도 점수 부여
- 총 위험도 기반 보상 배율 계산
- 난이도 구간: 0~3 쉬움, 4~6 보통, 7~9 어려움, 10+ 지옥
- 위험도 임계값으로 진화/보스/엔딩 조건 잠금 해제 가능

## 7. 메타 성장
- 재화: 정복의 잔재
- 획득원: 지역 클리어, 위험도, 보너스 맵, 업적
- 클래스별 3갈래 트리
- 노드는 선택형(전부 습득 불가)
- 수치 누적보다 플레이스타일 변화 중심

## 8. 스토리 전달
- 컷신 최소화
- 지역 시작/보스 처치 직후 짧은 텍스트+일러스트
- 마왕성에서 주요 서사 회수

## 9. 기술/구현 원칙 (Unity)
- 씬: Boot, MainMenu, Stage
- 런타임 상태: Loading -> Playing -> Paused -> Victory/Defeat -> Result
- 데이터 중심: ScriptableObject 사용
- 이벤트 중심 결합: 시스템 간 이벤트 버스 사용
- 성능 원칙:
  - 오브젝트 풀링 필수
  - 런타임 Instantiate 지양
  - 분산 Update 최소화
  - NonAlloc 충돌 처리 우선

## 10. 제작 파이프라인
1. Spec 작성
2. SO 데이터 입력
3. 프리팹 연결
4. 플레이테스트
5. 수치 조정
6. 변경 이력 기록
