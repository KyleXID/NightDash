# Production - QA & Playtest Protocol

## 1. 테스트 목적
- 기획 의도대로 재미 루프가 작동하는지 검증
- 밸런스/성능/안정성 이슈를 조기 발견

## 2. 테스트 트랙

## A. 기능 테스트
- 상태 전이
- 무기/패시브 레벨업
- 진화 조건
- 체크리스트 배율
- 세이브/로드

## B. 밸런스 테스트
- 클래스별 10회 플레이
- 빌드 선택률 기록
- 보스 도달률/클리어율 기록

## C. 성능 테스트
- 15분 풀런
- 피크 적/투사체 수 상황 측정
- FPS, GC Alloc 수집

## 3. 플레이테스트 시트 필수 컬럼
- Build Version
- Class
- Stage
- RiskScore
- Result (Win/Lose)
- TimeAlive
- LevelAtEnd
- TopWeapons
- DeathReason
- FunScore(1~5)
- PainPoint

## 4. 실패 원인 분류 코드
- D01: 순간 폭딜 사망
- D02: 기믹 피해 누적
- D03: 성장 정체
- D04: 시야 혼잡
- D05: 성능 저하로 조작 실패

## 5. 수용 기준 (MVP)
- 치명 버그(Crash/SoftLock) 0건
- 핵심 루프 블로커 버그 0건
- 보스 클리어율 25~45% 범위
- 특정 클래스 승률 2배 이상 편차 금지

## 6. 리그레션 체크리스트
- 레벨업 선택창 정상 노출
- 무기 슬롯 상한 정상
- 진화 1단/2단 조건 정상
- 체크리스트 보상 배율 정상
- 결과 보상 누락 없음

## 7. 테스트 주기
- 주 2회 내부 플레이테스트
- 주 1회 밸런스 리뷰
- 마일스톤 종료 시 종합 회귀 테스트
